# 【番外編・検討メモ】スキャルピング予測の手法

> 本連載（決算ファンダ × CAR、数日〜1ヶ月）とは**別レジーム**。数秒〜数分の値動きを取る。
> 効くシグナルも手法も変わる。**データと現実の壁が9割**を決める ― そこも正直に書く前提のメモ。

---

## 0. 大前提 ― データが手法を決める

予測対象は「次の数秒〜数分の中値（mid price）の方向／微小変化」。構造を与えているのは
**板（注文板＝Limit Order Book, LOB）と歩み値（約定の流れ）＝マイクロストラクチャ**。

- **必要データ**：ティック約定 ＋ **板（L2：気配の数量。できれば L3/MBO）**。日足・分足だけでは
  本質的シグナル（板の偏り）が取れない。
- **日本株の入手現実**：J-Quants は日次中心で板は弱い。実用にはブローカーAPIか有料ベンダー。
  → 詳細は §5。**ここが最初の関門**。

## 1. 特徴量（マイクロストラクチャ・シグナル）

- **オーダーブック・インバランス（OBI）／オーダーフロー・インバランス（OFI）**
  ― 短期方向予測で最も研究されている。Cont–Kukanov–Stoikov の OFI が定番。
- **マイクロプライス**（売買数量で重み付けした中値、Stoikov）
- **スプレッド・板の厚み・キュー位置**、**約定符号（tick rule）**、**VPIN**（情報トレード比率）
- 短期テクニカル：VWAP 乖離、短期ボラ、出来高プロファイル、約定の偏り

## 2. 手法（古典 → ML → DL → RL）

### A. 統計・計量（軽量・解釈しやすい／まずここから）
- **OBI/OFI → 次ティックの方向** を **ロジスティック回帰／線形回帰**。短期予測の "Hello World"。
- **Hawkes 過程（自己励起点過程）**：注文・約定が固まって来る性質をモデル化、到着強度・方向を予測。
- **GARCH 系**：短期ボラ予測（エントリー可否の判断に）。
- LOB を **マルコフ連鎖** で確率モデル化。

### B. 機械学習（教師あり：次の N バーの符号/変化を分類・回帰）
- **LightGBM／XGBoost** on マイクロ構造特徴量 ― 実務でよく使われる現実解。
- ランダムフォレスト、ロジスティック回帰（baseline）。
- ラベル付けは **トリプルバリア法（López de Prado）** で過学習を抑える。

### C. ディープラーニング（LOB 特化が研究の主戦場）
- **DeepLOB（Zhang–Zohren–Roberts, 2019）** ＝ CNN＋LSTM で生の板から中値変化を分類。**まず読む**。
- **LSTM/GRU、TCN（時系列CNN）、Transformer/attention**。
- ベンチマークは **FI-2010**（LOB 分類の標準データセット）。

### D. 強化学習（"予測" より "執行・建玉管理" の最適化）
- DQN/PPO 等で entry/exit・建玉サイズを最適化。スキャルピングの本質は予測＋**執行**なので相性は良い。
- ただし **非定常で過学習しやすく実運用は最難関**。

## 3. 現実の壁（必ず押さえる）

予測が当たっても勝てるとは限らない。

- **取引コスト**：スプレッド＋手数料＋スリッページ。利幅が小さいので
  コストがエッジを食う。**「コスト控除後に残るか」が唯一の指標**。
- **レイテンシ競争**：板の短期シグナルは既に **HFT が刈っている**。
  個人の秒〜ミリ秒の執行では、当たっても**約定できない／不利約定**。
- **非定常性**：時間帯・銘柄・地合いでマイクロ構造が変わり、モデルがすぐ陳腐化。
- **バックテストの罠**：look-ahead、甘い約定仮定。**リアルなフィルモデル**が無いと全部嘘になる。

## 4. 番外編にするなら（実験・教育トーン）

本連載の哲学（"AI は予測でなく発見"）に対し、
**「短期だけは構造があるので予測の余地がある／ただしエッジは薄く競争が激しい」** という
対比記事にすると面白い。最小パス：

1. 入手できる **分足/秒足＋簡易板** で OBI 風の特徴量を作る
2. **LightGBM で「次の N バーの方向」を分類** → コスト控除前の AUC／方向一致率
3. **コスト（スプレッド＋手数料）控除後** の損益曲線を主役に
   ― ここで「ほぼ消える」のが正直な結論になりやすい
4. 研究の到達点として **DeepLOB** を紹介し「本気でやるとこうなる」を見せる

## 5. データ入手の候補（日本株）

- **ブローカー API**：auカブコム **kabuステーション API**（板・約定が比較的取りやすい）、
  楽天 **MarketSpeed II RSS**、立花 **e支店 API**、SBI など
- **有料ベンダー**：QUICK、東証 **FLEX/raw feed**（本格的・高コスト）
- **J-Quants**：日次中心。板・ティックは弱い → スキャルピング用途には不足
- まず「**自分の口座でどの粒度（ティック/L2板/分足）が取れるか**」の確認が出発点

## 6. 参考キーワード／論文

- OFI: Cont, Kukanov, Stoikov, "The price impact of order book events"
- Microprice: Stoikov, "The micro-price"
- DeepLOB: Zhang, Zohren, Roberts (2019)
- ベンチマーク: FI-2010
- ラベリング/CV: López de Prado, *Advances in Financial Machine Learning*（triple-barrier, purged k-fold）
- 点過程: Hawkes process（注文到着のモデリング）
