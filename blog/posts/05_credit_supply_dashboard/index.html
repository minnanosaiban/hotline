信用需給ダッシュボードで「踏み上げ候補」を発見する ― 業績軸に需給軸を合流させる

「業績は良いのに株価が上がらない」「業績は悪いのに下がらない」 ― そんな違和感の背景にあるのが 信用需給 です。

連載01〜04 では業績軸を深掘りし、ＥＮＥＯＳ / 出光興産 / コスモエネＨＤ の構図を追ってきました。本記事ではそれに 需給軸 を合流させ、業績 × 需給の 4 象限マトリクスで「踏み上げ候補」と「踏み上げ罠」を可視化します。

信用需給の概要

ファンダメンタルズが優秀でも、信用買い残が多すぎれば戻り売り圧力で上値が重く、空売りが少なすぎれば材料が出ても急騰しません。逆に、業績が悪くても 空売り勢が積み上がっていれば踏み上げ で急騰することがあります。業績軸（連載01〜04）と需給軸（本記事）は 直交する 2 つの視点。両方を見ることで「上がる／下がる」の確率を立体的に評価できます。

需給を捉える中心指標が 信用倍率（信用残（買）÷ 信用残（売）） です。

信用倍率 状態 リスク
> 10 倍 空売りが極端に少ない 踏み上げ警戒 / 上方トリガーで急騰可能性
5〜10 倍 やや買い優位 中庸
1〜5 倍 拮抗 健全な需給
< 1 倍 空売り優位 弱気センチメント / 踏み上げ余地

ただし注意点があり、日本市場は信用倍率の中央値が 6.34 倍と構造的に買い優位（米国市場と異なり、貸株残高制限・空売り禁止銘柄の存在が影響）。「5 倍超で踏み上げ候補」を機械的に当てはめると候補が多くなりすぎるため、前週比（信用買い残の増減） や 出来高変化 など複数指標と組み合わせる必要があります。

そのうえで、業績軸（業績予想修正率）× 需給軸（信用倍率）の 4 象限マトリクス で銘柄を分類します。

業績NG（修正率 ≤ −3%） 業績OK（修正率 ≥ +3%）
踏み上げ寄り（信用倍率 > 5） ⚠️ 踏み上げ罠 ★ 踏み上げ候補
空売り優位（信用倍率 ≤ 1） 空売り正解 逆張り視点で要精査

最強ゾーンは右上 「業績OK × 踏み上げ候補」、最危険ゾーンは左上 「業績NG × 踏み上げ罠」。連載01〜04 の業績モメンタムだけでは見えなかった「需給の罠」が、このマトリクスで初めて可視化されます。

実装詳細は Appendix の Python コードを参照。

1. ＥＮＥＯＳ = 「業績NG × 踏み上げ罠」の継続中

連載02 で Consensus 22（下位）、連載03 で修正率 −3.7%（コンセンサス下振れ）、連載04 で経常変化率(予) −0.6%（わずかに減益予想）と、コンセンサスベースの業績悪化シグナルが積み重なってきました。それにもかかわらず信用買い残が今週 +3.3 万株増加。機械的に読めばこれは典型的な 踏み上げ罠 ですが、下方修正の主因が構造要因（のれん減損・在庫影響）であるため、個人が 構造を見抜いて仕込んでいる 可能性もあります（下記 callout 参照）。

📝 ＥＮＥＯＳ 信用買い増加は「罠」か「仕込み」か

「業績NG」の根拠であるコンセンサス下振れ（▲3.7%）の主因は のれん減損（非現金）・在庫影響（油価連動） ＝ 一時／構造要因。同時に JX金属 IPO（57.6% 売却）で当期利益 +1,300 億円・ネット D/E レシオ 0.46→0.40 改善という構造変化も進行中。

信用買い残が増加している背景には、個人投資家が 同じ構造を読み取って仕込んでいる 可能性があります。「踏み上げ罠」と読むか「構造を見抜いた仕込み」と読むかは、本業実態の解釈次第。

ENEOS 公式の「実質営業利益 4,400 億円維持」スタンス、4 つの修正率基準（▲94% 〜 +4.76% に分散） の試算、出典 PDF は 連載01 参照。

2. 出光興産 = 罠から脱出中？

ＥＮＥＯＳ と同じく業績シグナルは悪化（連載03 修正率 −3.5%）ですが、信用倍率 14.84 倍という極端な高水準 + 信用買い前週比 −10.7 万株という 急減。これは 信用買い玉の整理が進行している サイン ＝ 雪崩がすでに始まっており、整理が完了すれば需給がクリアになります。底値圏での反転シグナル の可能性があり、今後 1〜2 ヶ月の信用残推移を追う価値があります。

3. コスモエネＨＤ = 業績OK × 中庸需給

信用倍率 6.74 倍は市場中央値付近で過熱感なし。前週比 +12 千株の微増は健全。連載04 の「修正率・EPS超過プラス（経常データ欠損）」と合わせて、業績モメンタムが本物 + 需給がクリア という最も買いやすい状態です。

💡 同じ「業績NG」でも、信用買い残の 前週比 で「罠継続」か「整理進行中」かが判別できる。連続して残高推移を追うのが定石。

Streamlit アプリの紹介

本記事と同じ業績 × 需給 4 象限マトリクスを Streamlit + Plotly で操作可能にしたアプリを公開しています。信用倍率や修正率の閾値を動かしながら、「踏み上げ候補」と「踏み上げ罠」をリアルタイムで仕分けできます。

📝 アプリ公開予定: GitHub リポジトリ（準備中）。app_chart.py と同じく Streamlit ベースで、requirements.txt だけ揃えればローカルで動きます。

# Plotly 版 業績 × 需給 4 象限マトリクス（最小実装の抜粋）
import plotly.express as px

def fund_vs_credit_scatter(df, rev_th=3.0, cr_hot=5.0, cr_short=1.0):
    fig = px.scatter(df, x="業績予想修正率(予)", y="信用倍率",
                     log_y=True,  # 信用倍率は対数で見やすくなる
                     hover_data=["コード", "銘柄名", "信用残(買)", "前週比(買)", "ROE"],
                     color_discrete_sequence=["#7eaee0"])
    fig.add_vline(x=rev_th,  line_dash="dash", line_color="#27AE60")
    fig.add_vline(x=-rev_th, line_dash="dash", line_color="#E74C3C")
    fig.add_hline(y=cr_hot,   line_dash="dash", line_color="#27AE60")
    fig.add_hline(y=cr_short, line_dash="dash", line_color="#999999")
    fig.update_layout(xaxis_title="業績予想修正率(%) ← 下振れ  上振れ →",
                      yaxis_title="信用倍率（対数）  ← 空売り優位  踏み上げ →")
    return fig

データは連載中の classify_fund_vs_credit() を流用、4 象限ごとに色分けして表示できます。

Appendix ― Python コード

画像生成の全コードは 05_credit_dashboard_make_images.py を参照。執筆者ローカルのモジュール・データに依存するため、そのままでは動きません。

import pandas as pd

# 信用倍率 = 信用残(買) ÷ 信用残(売)
# しきい値（業績 × 需給 4 象限分類）:
#   業績OK × 踏み上げ候補:  業績予想修正率(予) ≥ +3% かつ 信用倍率 > 5
#   業績OK × 空売り優位:    業績予想修正率(予) ≥ +3% かつ 信用倍率 ≤ 1
#   業績NG × 踏み上げ罠:    業績予想修正率(予) ≤ -3% かつ 信用倍率 > 5
#   業績NG × 空売り正解:    業績予想修正率(予) ≤ -3% かつ 信用倍率 ≤ 1
# 業績変化 ±3% は連載03 と統一、信用倍率 5 倍は日本市場の中央値（6.34 倍）付近
# 流動性フィルタ: 時価総額 ≥ 100 億円 / ROE ≥ 5%

def compute_credit_ratio(df: pd.DataFrame) -> pd.Series:
    """信用残(買) ÷ 信用残(売) で信用倍率を計算。信用残(売)=0 の銘柄は NaN。"""
    return df["信用残(買)"] / df["信用残(売)"].where(df["信用残(売)"] > 0)


def classify_fund_vs_credit(df: pd.DataFrame,
                            rev_col: str = "業績予想修正率(予)",
                            cr_col: str = "信用倍率") -> pd.Series:
    """業績軸と需給軸の組み合わせで 4 象限分類。"""
    zone = pd.Series("中立", index=df.index)
    ok = df[rev_col] >= 3
    ng = df[rev_col] <= -3
    hot = df[cr_col] > 5
    short = df[cr_col] <= 1

    zone[ok & hot]    = "業績OK × 踏み上げ候補"
    zone[ok & short]  = "業績OK × 空売り優位"
    zone[ng & hot]    = "業績NG × 踏み上げ罠"
    zone[ng & short]  = "業績NG × 空売り正解"
    return zone

データ出典: 証券会社が無料で提供する銘柄情報サービスから取得した CSV 6 指標（信用残(買)/信用残(売)/前週比(買)/前週比(売)/信用残/売買高レシオ/業績予想修正率(予) + ROE/時価総額のフィルタ用 2 指標）
