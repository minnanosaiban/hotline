セグメント発進力スコア ― 決算短信セグメント情報で「次の主力事業」と「隠れた高収益事業」を発掘する

連載04〜07 で扱った証券会社が無料で提供するデータは、企業を 1 つの数字で見ます。「PER」「ROE」「業績予想修正率」 ― どれも全社合算の指標です。しかし企業の本当の収益構造は 事業セグメント別 にしか見えません。

連載06 で構築した自前 XBRL パイプラインを使うと、ソニーグループの 6 セグメント に色付け、アドバンテストのテストシステム事業の営業利益率 50.9%（売上 1.02 兆円）、マキタ アジア事業の利益率 88.4% といった、無料で取れるデータには絶対に出てこない数値が一発で取り出せます。

本記事では決算短信 XBRL から 233 銘柄 × 787 セグメント の前期比成長率と利益率を分析し、「次の主力事業」と「隠れた高収益事業」を機械的に発掘します。

セグメント分析の概要

連載01〜11 では届かなかった視点

連載01〜11 で扱ったすべての指標は 企業単位 でした。しかしファンダメンタル分析の本場では、セグメント別の分析が当然の前提 です。

連載 主要指標 視点
01-05 PEG / ROE / リビジョン等 企業合算
09 進捗率 Z-score 企業合算（売上・営利）
10 アクルーアル 企業合算（純利益・営業CF）
11 三角検証 企業合算（EPS）
12 セグメント セグメント別 成長率 × 利益率 事業ポートフォリオ

「PER 20 倍」「ROE 10%」が企業全体の数値だとしても、裏で何の事業が伸びていて、何が縮んでいるか は分かりません。総合電機メーカーや商社のような複合企業ほど、セグメント分析の価値は大きくなります。

セグメント発進力スコアの設計（本記事の簡略版）

機関投資家が使う「セグメント発進力スコア」の本来形は：

発進力スコア = 直近 1 年の成長率 ÷ 過去 5 年平均 CAGR
              ・1.5 以上 → 加速中
              ・0.7 以下 → 減速中

本記事では 決算短信 XBRL の時系列が 2 年分しかない ため、簡略化して 「前期比成長率」 で代用します。次回以降の連載で蓄積が進めば、本来の 5-7 年スパンの発進力スコアに移行します。

セグメントデータのカバレッジ状況

連載06 で構築した自前パイプラインの現状：

データソース セグメント有り
有報 XBRL（13 銘柄 × 7 期 = 91 ファイル） 0 件（parser_version 0.2.0 で yuho セグメント未対応）
決算短信 XBRL（1,624 ファイル） 634 件 / 銘柄数 400
うち 2 年分時系列を持つ銘柄 233 銘柄

連載01〜11 の主役 ＥＮＥＯＳ は本記事用に追加取得し、2026/3 期通期の決算短信 XBRL から 5 セグメントの内訳が分析可能 になりました（後述の「ＥＮＥＯＳ のピークアウト内訳」参照）。出光 / コスモエネＨＤ は同期未取得のため引き続き対象外。これに加え、連載05 で登場した主要 6 社のうち ソニーＧ（6758）・トヨタ（7203） や、連載10-11 で扱った 総合商社の丸紅（8002）・双日（2768） が決算短信 XBRL でカバーされており、本記事の narrative の中心になります。

本記事の実装スコープ

本記事で扱うこと:
  ・決算短信 XBRL 233 銘柄から 787 セグメントの前期比成長率を計算
  ・売上 100 億円以上で絞った加速 Top10 / 減速 Worst10
  ・売上 200 億円以上で絞った高営業利益率セグメント
  ・主要 4 銘柄（ソニーG/トヨタ/信越化/任天堂）の 2 年詳細
  ・総合商社（丸紅・双日）の事業転換シグナル
  ・ＥＮＥＯＳ 2026/3 期 5 セグメントのピークアウト内訳

本記事で扱わないこと:
  ・5-7 年スパンの本格的な発進力スコア（蓄積後の課題）
  ・有報からのセグメント抽出（parser 拡張が必要）
  ・セグメント別 EVA・WACC（フェーズ4 で扱う）

連載05 のマルチファクター分析で 総合スコア 44.1 と中庸評価だったソニーG。決算短信 XBRL のセグメント情報を取り出すと、事業ポートフォリオの分散 が一気に見えてきます。

音楽事業の営業利益率 19.6% が突出。売上規模ではゲームの 40% 程度ですが、利益率は 2 倍以上。Spotify / Apple Music が世界を席巻するストリーミング時代でも、音楽パブリッシング・原盤権という 「軽資産で永続キャッシュフロー」 ビジネスの収益性が見えます。

連載05 の総合スコア 44 はこの 「音楽 + イメージング・センサ」の高利益率セグメントを企業合算で平均化したため過小評価 していた可能性が示唆されます。セグメント別に見ることで、ソニーG ≠ 単なるゲーム会社 ≠ 単なるエレキ会社 という事業ポートフォリオの強み が浮かびます。

主要 4 銘柄の 2 年セグメント推移

連載01〜11 で登場した主要銘柄について、2 年分のセグメント別売上推移を確認します。

ＥＮＥＯＳ のピークアウト内訳 ― 連結 +25.5%（公表）/ +339.8%（継続事業ベース）急回復の裏にある OPM 低下

本連載の中核銘柄である ＥＮＥＯＳ について、2026/3 期通期の決算短信 XBRL を取得し、主要 6 セグメントの当期 vs 前期を比較しました。連結営業利益は 公表ベース 前期 3,717 億円 → 当期 4,666 億円（+949 億、+25.5%）、JX金属を非継続事業として除外した 継続事業ベースでは 前期 1,061 億円 → 当期 4,666 億円（+339.8%） という急回復を示しました。さらに同社開示の 「在庫影響除き営業利益相当額 4,744 億円」 は、2025/3 期業績予想修正発表時の主張（実質営業利益 4,400 億円維持）を 上回って着地 しています（連載04 4 基準試算参照）。セグメント別に分解すると、この急回復の構造が複層的に見えてきます。

連結業績の急回復（公表 +25.5% / 継続事業ベース +339.8%）の正体は、売上の 92% を占める石油製品セグメントが前期赤字 (−507 億) から当期黒字 (+2,924 億) に転換 したことに加え、その他事業の 912 億円（金属 442 億 + NIPPO・連結調整 470 億）が連結利益の約 20% を貢献していることです。在庫評価差益や精製マージン回復など、原油価格サイクルに連動する要素が大きく寄与しています。連載09 で見た通り、石油元売の在庫影響は単年では大きく揺れますが 2022-2024 累積では CF が純利益を上回って回収済み ― 在庫影響そのものは「一過性」ではなく 業態固有の上下サイクル の一部です。

一方で、本来の収益エンジンである 2 セグメントの利益率は低下 しています:

  • 石油・天然ガス開発: 5 セグメント中で唯一 OPM 20% 超だった高収益事業。OPM 36.00% → 23.45%（−12.54pt）、営業利益 −41.8%。ただし OPM 23% は依然として他業界比較で高水準で、原油・ガス価格高止まり期からの サイクル正常化 とも読めます。「構造的剥落」か「サイクル要因」かは 2 年比較では断定できず、継続観察が必要
  • 機能材: 「高付加価値素材で利益を安定化」と説明される事業だが、OPM 5.09% → 3.27%（−1.82pt）、営業利益 −37.3%。化学市況の悪化が利益率を圧迫している点は注意が必要

この 「連結回復 vs 高 OPM セグメントの正常化」の二層構造 は、連載で観察されてきた ENEOS narrative の複層性を裏付けます。

連載01-02: GARP マップで割安・低 ROE。GARP 圏外でも +29.7% 上昇
連載03   : 有報 XBRL で 2022 純利 5,371 億（ウクライナ侵攻特殊年）→ 2025 2,261 億 縮小
連載09   : 2022 単年 CF/純利 39% vs 3 年累積 114%（回収済み）
連載10   : EPS +72.9% vs 経常 ▲0.6% の併存は構造要因（のれん減損・JX金属IPO・自社株買い）で説明可能
連載12   : 2025-03-28 業績予想修正発表 CAR [-1,+20]=-13.89%（主因は のれん減損・在庫影響 = 構造要因）
連載11   : セグメント別で「開発 OPM 36→23%、機能材 OPM 5→3%」と高 OPM セグメントの正常化局面を可視化、連結 +25.5%（公表）/ +339.8%（継続事業ベース）/ 4,666 億、在庫除き 4,744 億 で主張通り着地 ← 本記事

連載12 までは 「全体としてピークアウト」 が観察対象でしたが、本記事のセグメント分解で 「主力 2 事業の OPM 低下と石油製品の急回復が同時進行している」 という複層構造が見えてきます。連結 +542% の表面に騙されず、「石油製品の在庫評価差益が連結を押し上げた裏で、高 OPM セグメントは正常化局面に入っている」 という多面的な解釈が、決算短信 XBRL のセグメント情報から定量的に裏付けられました。「ピークアウトか正常化か」は連載09 で見た CF 累積（CF/純利 114%） やセグメント時系列の継続観察と併せて判断する必要があります（ENEOS 自身は「実質営業利益 4,400 億円水準を維持」と主張、連載04 4 基準試算参照）。

なお出光興産 / コスモエネＨＤ は本記事執筆時点で同期 (2026/3 期) の決算短信 XBRL を未取得です。3 社揃えれば「石油元売 3 社の事業構成比較」が可能になり、連載04〜07 で観察された 3 社の株価動向の差をセグメントレベルで説明できる見込みです。

セグメント分析の計算方法

1. 連載06 スキーマからのセグメント抽出

連載03 で設計したスキーマに従い、決算短信 JSON の segments.current リストから時系列を構築します。

import json
from pathlib import Path

def load_segment_timeseries(stmts_dir: Path = Path("data/statements")):
    """銘柄 × 会計年度 × セグメントリスト の辞書を構築。"""
    by_code = {}
    for f in stmts_dir.glob("*_FY.json"):
        if "forecast" in f.name:
            continue
        d = json.load(open(f, encoding="utf-8"))
        meta = d.get("metadata", {})
        seg = d.get("segments", {})
        cur = seg.get("current", []) if isinstance(seg, dict) else []
        if not cur:
            continue
        code = meta.get("code")
        fy = meta.get("fiscal_year_end")
        if code and fy:
            by_code.setdefault(code, {})[fy] = cur
    return by_code

連載03 スキーマの segments.current リスト構造 が、セグメント間の比較を直感的にしてくれます。各 dict には key（セグメント識別子）、label（日本語名）、net_sales / operating_income などが入っています。

2. 複数の値候補から取り出す

セグメント XBRL では同じ「売上」を表す要素 ID が複数あります（net_sales / external_revenue / total_revenue）。マッピング辞書で吸収しきれなかった部分を Python 側で複数候補リスト から拾います。

def get_segment_value(s: dict, keys: list[str]) -> float | None:
    """セグメント dict から複数候補キーで値を取り出す。"""
    for k in keys:
        v = s.get(k)
        if isinstance(v, (int, float)):
            return v
    return None


# 使用例
sales = get_segment_value(seg, ["net_sales", "external_revenue", "total_revenue"])
oi = get_segment_value(seg, ["operating_income", "segment_profit"])

3. 前期比成長率の計算

def compute_yoy_growth(by_code: dict) -> pd.DataFrame:
    """全銘柄でセグメント前期比成長率を計算。"""
    rows = []
    for code, fy_map in by_code.items():
        yrs = sorted(fy_map.keys())
        if len(yrs) < 2:
            continue
        prev_yr, cur_yr = yrs[-2], yrs[-1]
        prev_map = {s.get("key"): s for s in fy_map[prev_yr]}
        for s in fy_map[cur_yr]:
            key = s.get("key")
            if key not in prev_map:
                continue
            cur_ns = get_segment_value(s, ["net_sales", "external_revenue", "total_revenue"])
            prv_ns = get_segment_value(prev_map[key],
                                       ["net_sales", "external_revenue", "total_revenue"])
            if cur_ns is None or prv_ns is None or abs(prv_ns) < 1e8:
                continue
            rows.append({
                "code": code,
                "segment": s.get("label") or key,
                "sales_growth": (cur_ns - prv_ns) / abs(prv_ns) * 100,
                "cur_sales_oku": cur_ns / 1e8,
            })
    return pd.DataFrame(rows)

4. 加速判定のしきい値

5-7 年蓄積が無いため、本記事では暫定しきい値を使います。

前期比成長率 判定
≥ +50% 著しい加速（新規事業の本格寄与・M&A）
+20% 〜 +50% 明確な加速
−20% 〜 +20% 安定
−20% 〜 −50% 明確な減速
< −50% 著しい減速（事業撤退・大型減損）

蓄積が 5 年以上になれば、本来の 「直近 1 年成長率 ÷ 過去 5 年 CAGR」 で判定できます。例えば直近 1 年が +20% で 5 年 CAGR が +5% なら、発進力スコア = 20 / 5 = 4.0 で「明らかな加速」と判定できる。

5. 投資判断への組み込み

新規投資前のセグメントチェック:
  ① 主力セグメント（売上構成 50% 以上）の成長率 ≥ 0
  ② 高利益率セグメント（利益率 ≥ 20%）が存在
  ③ 減速セグメントの売上構成 ≤ 30%

事業転換期の発見:
  ・主力減速 × 新規加速 の組み合わせ（例: 丸紅）
  ・新規セグメントの利益率が主力を上回る

高利益率セグメント発掘

def find_high_margin_segments(dfg: pd.DataFrame,
                               min_sales_oku: float = 200,
                               min_margin: float = 30) -> pd.DataFrame:
    """売上 200 億以上で営業利益率 30% 超のセグメントを抽出。"""
    sub = dfg.copy()
    sub = sub[sub["cur_sales_oku"] >= min_sales_oku]
    sub = sub.dropna(subset=["cur_oi_oku"])
    sub["margin"] = sub["cur_oi_oku"] / sub["cur_sales_oku"] * 100
    return sub[sub["margin"] >= min_margin].sort_values("margin", ascending=False)

事業転換期の検出

def find_transition_companies(dfg: pd.DataFrame,
                               growth_threshold: float = 50,
                               decline_threshold: float = -20) -> pd.DataFrame:
    """同一銘柄内に加速セグメントと減速セグメントが共存する企業を抽出。"""
    by_code = dfg.groupby("code")
    transition = []
    for code, group in by_code:
        has_acc = (group["sales_growth"] >= growth_threshold).any()
        has_dec = (group["sales_growth"] <= decline_threshold).any()
        if has_acc and has_dec:
            transition.append({
                "code": code,
                "max_growth_segment": group.loc[group["sales_growth"].idxmax(), "segment"],
                "max_growth": group["sales_growth"].max(),
                "min_growth_segment": group.loc[group["sales_growth"].idxmin(), "segment"],
                "min_growth": group["sales_growth"].min(),
            })
    return pd.DataFrame(transition)


# 使用例: 丸紅のような事業転換期銘柄を抽出
transition = find_transition_companies(dfg, growth_threshold=50, decline_threshold=-20)

連載09 アクルーアル × 連載11 セグメントのクロス

def combine_accrual_with_segment(accrual_df: pd.DataFrame,
                                  segment_df: pd.DataFrame) -> pd.DataFrame:
    """連載09 のアクルーアル × 連載11 のセグメント前期比を結合。

    健全アクルーアル + 主力セグメント加速 = 最強買い候補
    """
    # 銘柄別の主力セグメント（最大売上）の成長率を取得
    primary = segment_df.sort_values("cur_sales_oku", ascending=False) \
                       .groupby("code").head(1)[["code", "segment", "sales_growth"]] \
                       .rename(columns={"segment": "primary_segment",
                                        "sales_growth": "primary_growth"})

    merged = accrual_df.merge(primary, on="code", how="left")
    merged["composite_score"] = (
        # アクルーアル健全度（負ほど高得点）
        np.clip(-merged["mean_accrual"] * 500, 0, 40)
        # 主力セグメント成長
        + np.clip(merged["primary_growth"], -50, 100) * 0.3
    )
    return merged.sort_values("composite_score", ascending=False)
